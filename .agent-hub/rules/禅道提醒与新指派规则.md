# 禅道提醒与新指派规则

来源：`/Users/se7en/000group/禅道任务2.0/禅道任务提醒`  
原产物作者：`lihualing`

## 到期提醒

- 默认提醒范围：逾期、今天到期、明天到期。
- 任务必须有有效截止日才进入到期判断。
- “今天不再提醒”的去重键为：任务 ID、截止日、当天日期。
- “稍后提醒”应使用一次性提醒，不得无限循环拉起弹窗。
- 如果没有需要提醒的任务，默认安静退出。

## 新指派提醒

- 首次运行时只建立已见任务基线。
- 后续运行时，新增任务 ID 且指派给当前账号，视为新指派。
- 新指派提醒展示成功后再写入已见记录。
- 不重复提醒已经看过的任务。

## 缓存和失败处理

- 执行列表可以缓存，原工具默认缓存 360 分钟。
- 最终任务结果可按短 TTL 缓存，减少重复请求。
- 禅道连接失败时，可回退到最近一次有效缓存。
- 使用缓存时必须在摘要中说明“数据可能不是最新”。

## 弹窗稳定性

- 不得同时打开多个提醒窗口。
- 同一轮多条任务应合并展示或串行展示。
- 用户关闭、取消或脚本异常时必须退出当前流程。
- 窗口尺寸和布局应稳定，长内容用滚动区承载。

## 工作台迁移建议

- Windows 计划任务可作为历史实现参考，不建议原样搬到工作台。
- 工作台应使用自身调度、数据库状态和通知层实现提醒。
- 提醒状态建议持久化到工作台配置或用户维度状态表。
