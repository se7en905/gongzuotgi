# 禅道个人任务工作台

来源：`/Users/se7en/000group/禅道任务2.0/禅道任务提醒`  
原产物作者：`lihualing`

## 适用场景

- 用户要查看自己当前未完成的禅道任务。
- 用户要按逾期、今天、明天、即将到期、全部任务梳理工作。
- 用户要识别新指派给自己的任务。
- 用户要生成工作台任务摘要，而不是直接打开 Windows 版看板。
- 用户要从任务卡片进入打开任务、登记工时、备注、复制标题、生成美术简报等动作。

## 输入

优先接受以下输入：

- 禅道账号，未提供时使用当前登录账号。
- 任务状态范围，默认 `wait`、`doing`、`pause`。
- 执行状态范围，默认 `doing`、`wait`。
- 截止范围，默认关注逾期、今天、明天。
- 是否强制刷新缓存。

## 任务抓取规则

1. 通过禅道账号读取当前指派给我的任务。
2. 扫描执行列表时默认只看进行中和未开始执行。
3. 任务状态默认保留 `wait`、`doing`、`pause`。
4. 排除已删除任务。
5. 对子任务要递归收集，不能只取顶层任务。
6. 按截止日期升序排序；同一天内按任务 ID 升序排序。
7. 输出字段至少包含：
   - 任务 ID
   - 任务标题
   - 状态
   - 优先级
   - 截止日
   - 预计开始
   - 指派时间
   - 执行 ID
   - 执行名称
   - 关联需求 ID
   - 关联需求标题
   - 预计工时
   - 已耗工时
   - 剩余工时

## 看板分组

按负责人视角优先分组：

- 逾期：截止日早于今天。
- 今天：截止日等于今天。
- 明天：截止日等于明天。
- 即将到期：截止日在今天到配置天数范围内。
- 全部：所有未完成任务。

输出时优先给风险和行动建议：

1. 逾期任务数量和清单。
2. 今天必须处理的任务。
3. 明天需要提前协调的任务。
4. 新指派但未处理的任务。
5. 没有截止日或字段异常的任务。

## 新指派提醒规则

- 首次运行时记录当前任务 ID 集合作为已见任务。
- 后续运行时，如果发现新任务 ID 且指派给当前账号，则列为新指派。
- 新指派提醒只提示新增项，不重复提示已见任务。
- 成功展示后再更新已见任务集合。
- 如果禅道连接失败，允许读取最近一次有效缓存，但必须提示数据可能不是最新。

## 到期提醒规则

- 默认关注今天、明天和已逾期任务。
- 用户点击“今天不再提醒”后，只对当前任务 ID、截止日、当天日期组合生效。
- 稍后提醒应创建一次性提醒，不得使用无限循环。
- 同一轮出现多条提醒时必须串行处理或合并展示，不能并发弹窗。

## 任务动作

工作台可以把以下动作设计为按钮或菜单：

- 打开：打开禅道任务详情。
- 工时：打开登记工时页面。
- 备注：打开任务编辑或备注页面。
- 复制标题：复制任务标题。
- 美术：调用 `zentao-art-brief` 生成美术简报。

## 输出格式

默认输出中文摘要：

```md
## 禅道个人任务看板

- 更新时间：
- 当前账号：
- 扫描范围：

### 风险概览
- 逾期：
- 今天到期：
- 明天到期：
- 新指派：

### 任务清单
| ID | 状态 | 截止日 | 优先级 | 剩余/已耗 | 执行 | 标题 |
|---|---|---|---|---|---|---|

### 负责人建议
- ...
```

## 禁止事项

- 不得因为读取任务而修改禅道任务字段。
- 不得顺手关闭、完成、改截止日、改工时或改指派人。
- 不得隐藏禅道连接失败、缓存命中、字段缺失等风险。
- 不得把 Windows 计划任务安装脚本直接迁移到工作台执行流程；工作台应实现自己的调度和状态持久化。
