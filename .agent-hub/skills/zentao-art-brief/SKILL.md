# 禅道美术简报

来源：`/Users/se7en/000group/禅道任务2.0/禅道美术简报`  
原产物作者：`lihualing`

## 适用场景

- 用户给出禅道任务 ID 或任务链接，希望快速知道美术要做什么。
- 用户要从开发需求中拆出美术产出项、排除项和待确认项。
- 用户要查看任务描述、关联需求、评论里的参考图。
- 用户是美术部门负责人，需要判断资源投入、风险、是否需要追问产品或开发。

## 输入

- 禅道任务 ID，例如 `48848`。
- 禅道任务 URL，例如包含 `taskID=48848` 的链接。
- 可选：是否下载图片附件。
- 可选：输出目录或报告路径。

## 读取范围

读取禅道任务后，应综合以下内容：

- 任务标题。
- 任务描述。
- 关联需求描述。
- 需求验收标准。
- 任务评论和补充说明。
- 图片附件和正文图片。

## 美术识别关键词

优先识别包含以下语义的内容：

- 美术、效果图、UI、图标、金币、蒙版、提示条、tips、弹层、按钮、卡片。
- 展示、样式、切图、资源、视觉、客户端、web。
- 锁住、锁定、未解锁、门槛、奖励展示、尺寸、格式、大小限制、间距、字号、Banner、图片、短文案。

## 开发/后台排除关键词

以下内容通常不作为美术产出：

- 接口、后台、校验、发放、派发、字段、保存、回显、自动、逻辑、数据库、统计口径。
- 配置、启停、启用、关闭、开启、排序、跳转、内链、外链、上传、维护、选择、开关、运营。
- 鼠标移入、鼠标移出、预览按钮、点击弹出预览等后台交互配置。

如果同一句同时包含明确视觉交付要求，例如图标尺寸、展示尺寸、效果图、视觉样式，则不能直接排除，需要保留为待确认或美术产出。

## 产出分类

### 需要美术做

收集明确要求美术提供或确认的内容：

- 新增图标、默认图、入口图、奖励图。
- 弹窗、卡片、按钮、提示条、蒙版、锁住态等视觉状态。
- 尺寸、格式、间距、字号、Banner、展示规则中的视觉部分。
- 需要补齐的新增入口默认图标。

### 不需要美术做

收集明确的“不做”或非美术内容：

- 原文包含不要、不需要、不调整、不做、无需、不再。
- 后台配置、接口逻辑、字段保存、数据口径。
- 被识别为后台/表格类截图的图片，不放入主参考图区。

### 需要确认

收集负责人需要追问的内容：

- 金币奖励是否沿用现有图标、单位和领取反馈。
- 锁住态覆盖范围是局部区域还是整个模块。
- 视觉效果缺少明确尺寸、状态或适配要求。
- 任务描述中存在美术和开发边界不清的语句。

## 图片规则

1. 从任务正文和附件中提取 `png`、`jpg`、`jpeg`、`gif`、`webp`。
2. 图片链接要解析为可访问的禅道文件地址。
3. 参考图优先展示客户端、游戏、Web 前台视觉截图。
4. 疑似后台表格、配置页面、低色彩后台截图应放入“已过滤”区域，不作为主参考图。
5. 图片下载失败时保留 URL 和失败原因。

## 输出格式

默认输出中文报告：

```md
# 禅道美术简报

- 任务：
- 状态：
- 截止：
- 指派：
- 关联需求：
- 原产物作者：lihualing

## 需要美术做
- ...

## 不需要美术做
- ...

## 需要确认
- ...

## 参考图
- ...

## 美术相关原文
...
```

## 负责人判断口径

- 先判断是否需要新增资源，再判断是否只是沿用现有资源。
- 对“展示规则”只抽取视觉相关部分，不把后台配置当成美术工作。
- 对“不做”描述要保留原意，避免误派给美术。
- 对需求边界不清的内容，输出为“需要确认”，不要替业务拍板。

## 禁止事项

- 不得修改禅道任务内容、需求内容或附件。
- 不得伪造参考图或补全原文没有的信息。
- 不得把后台配置类工作误归为美术产出。
- 不得删除原始图片链接和识别依据。
