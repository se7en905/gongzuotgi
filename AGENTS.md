# 项目Agent开发基线

## 1.项目信息、技术栈版本

- 项目路径：`/Users/se7en/ArtProject/platform`
- 项目定位：美术部门生产工作台，用于禅道任务/BUG同步、任务中心、AI产物清单、成员产物、质量评分、资料库库存与负责人管理操作。
- 默认用户身份：美术部门负责人。涉及任务、排期、周报、资源协调、风险识别、跨团队协作时，优先按负责人视角组织方案和回复。
- 默认回复语言：中文。
- 用户可见文案、说明文档、标记文档、看板文案统一使用中文。代码标识、命令、路径、文件名、字段名、接口参数、第三方专有名词可保留英文。
- 前端技术栈：
  - Vue 3
  - Vite 5
  - Element Plus
  - Sass
  - Node.js `>=20`
- 后端技术栈：
  - Node.js ESM
  - `mysql2`
  - 本地 JSON/缓存文件
  - macOS LaunchAgent
- Git 远端：
  - `origin`：`https://github.com/se7en905/ArtPlatform.git`
  - `platform-backup`：`https://github.com/se7en905/gongzuotgi.git`
- Git 自动备份：
  - 由本机 macOS LaunchAgent 定时执行，不经过 Codex 对话，不调用 AI，不消耗 Codex/OpenAI token。
  - 默认每 30 分钟检查一次，有变更才自动提交并推送。
  - 自动备份远端固定为 `platform-backup/main`。
  - 自动备份脚本必须只依赖本机 Git/GitHub 凭据，运行时不得使用 AI、Codex 或 OpenAI 环境变量。

## 2.命名、组件、样式、目录规范

- 保持现有目录和模块边界，不得为了局部修复顺手重构、改名、换布局或调整业务结构。
- 关键文件职责：
  - `server/scanner.mjs`：Git、本地路径、共享盘扫描与产物内容解析。
  - `server/server.mjs`：平台接口、缓存读取、禅道同步入口、权限判断。
  - `src/App.vue`：库存缓存读取、刷新库存、贡献人覆盖、AI评分、产物使用统计等全局业务状态。
  - `src/views/SkillInventoryView.vue`：AI产物清单、贡献人调整、使用明细抽屉、成员产物整合入口。
  - `src/components/dialogs/RunCreateDialog.vue`：新建美术执行弹窗。
  - `src/styles.scss`：全局展示兜底样式、抽屉、评分面板、弹窗分段控件样式。
  - `scripts/git-auto-backup.sh`：Git自动备份执行脚本。
  - `scripts/install_git_auto_backup_launch_agent.sh`：Git自动备份 LaunchAgent 安装脚本。
- 工作台不是营销页或展示页。UI设计优先保证数据准确、信息完整、操作稳定、密度合理。
- 工作台界面关键业务文本不得出现无意义省略：
  - 按钮
  - 分段控件
  - 标签
  - 抽屉标题
  - 成员名
  - 产物名
  - 版本记录
  - 任务标题
  - 操作字段
- 长路径、日志、表格大段文本可以受控换行或提供详情入口，但不能让负责人无法判断真实内容。
- 列表操作栏规则：
  - 任务中心操作栏固定在右侧。
  - AI产物清单操作栏也必须固定在右侧。
  - 响应式宽度变化时操作字段不得消失。
  - 操作按钮必须左右横向排列，不得换行导致按钮不可见。
- AI产物清单按钮规则：
  - 负责人可见：`明细`、`作废`。
  - 已作废后按钮显示为：`恢复`。
  - 组员只可见：`明细`。
- AI产物清单版本展示规则：
  - 单条产物被作废后，版本展示为置灰的 `1.0`。
  - 负责人点击恢复后，该产物版本恢复为正常版本展示。
  - 作废和恢复只影响工作台展示、评分计入和状态，不修改源文件。
- 本地路径和共享盘扫描产物展示规则：
  - 产物列表只显示文件夹名称或负责人配置的展示名称。
  - 路径扫描出的文件夹产物不得显示绿色小字说明框、绿色角标或额外来源说明。
  - 展示管理界面必须支持全选。
  - 展示管理界面必须支持勾选是否展示；取消展示后，该产物不出现在 AI产物清单。
  - 展示管理界面必须允许负责人修改展示名称或产物名称。
  - 展示管理界面必须允许负责人为路径扫描产物填写产物名称；填写后主产物列表直接显示该名称。
  - 展示管理界面必须允许负责人为路径扫描产物维护调用别名。
  - 展示管理界面必须允许负责人通过下拉选项选择贡献人，且贡献人支持多选。
  - 展示管理界面必须允许负责人通过下拉选项选择类型；当前类型选项为 `Skill`、`规范` 和 `文件夹产物`。
  - 展示管理里选择类型只影响工作台展示、统计分类和质量分入口，不修改本地路径、共享盘或 Git 源文件。
- 接入扫描弹窗规则：
  - 弹窗内只负责选择来源并开始扫描，来源为 Git 仓库、本地路径、共享盘路径三选一。
  - 弹窗内不放 `刷新库存` 入口；总界面保留统一 `刷新库存` 入口。
  - 已接入来源、来源名称、上次扫描状态和上次扫描内容，默认展示平台缓存，不因打开弹窗而实时刷新。
- 全局右侧半屏抽屉：
  - 右侧半屏抽屉。
  - 必须为直角表现。
  - 所有通过 Element Plus `ElDrawer` 从右侧滑出的界面都必须全局直角，包含抽屉本体、头部和内容区。
  - 不得在后续样式中用 `.el-drawer.rtl`、`.art-progress-log-drawer`、`.skill-usage-drawer` 或类似选择器恢复圆角。
  - 布局紧凑。
  - 不恢复绿色说明框、版本角标或已删除解释文案。
  - 通过 `append-to-body` 渲染时，样式必须同时考虑全局样式和组件内 scoped 样式。
- “新建美术执行”弹窗：
  - 执行来源和执行范围分段控件必须使用全倒角选中底图。
  - 文字必须完整可见。
  - 不得改变已确认交互流程。
- 拖拽规则：
  - 任务拖拽必须稳定。
  - 任务只要在对应成员身上，就必须能拖拽成功。
  - 拖拽稳定性与禅道同步无关，不能因同步状态影响本地拖拽成功率。
- 本地弹窗、提醒、LaunchAgent、轮询脚本：
  - 不得造成 Dock 反复闪烁。
  - 不得抢焦点。
  - 不得重复弹窗或窗口反复出现/消失。
  - 不得使用无限重弹的 `while true` 弹窗逻辑。
  - 同一轮多条待处理事项必须串行弹窗或写入待弹队列。
  - JXA/Cocoa 自定义窗口默认使用 `NSApplicationActivationPolicyAccessory`。
  - 弹窗必须固定尺寸和稳定布局，长内容使用滚动区域、列表或摘要窗口。

## 3.接口、数据结构、数据库约定

- 禅道同步范围：
  - 只看部门 ID `27`。
  - 精准抓取美术部门人员身上的任务。
  - 任务和 BUG 必须作为独立队列分开同步。
  - 任务同步失败不得影响 BUG 队列，BUG 同步失败不得影响任务队列。
- 禅道字段保护：
  - 未经负责人明确要求，不得修改或清空父任务、关联需求、任务描述、标题、截止日、模块、任务类型、状态、预计工时、剩余工时等原字段。
  - 如确实需要改这些字段，必须由负责人明确要求并二次确认。
- 禅道连接方式：
  - 当前以主动抓取/手动刷新同步为主。
  - 长期更稳的方案可以接禅道 Webhook 或增量变更接口，但前提是禅道侧支持。
  - 在禅道侧无法稳定提供 Webhook/增量接口前，必须把现有主动抓取方案做稳定。
- 禅道同步按钮：
  - 任务大厅按钮文案为 `同步任务`，只触发任务同步和任务列表刷新。
  - Bug 大厅按钮文案为 `同步 Bug`，只触发 Bug 同步和 Bug 列表刷新。
  - 除非负责人明确要求合并同步，不得把任务和 Bug 同步重新绑在同一次点击里。
  - 普通点击 `同步任务` 必须走快速任务同步：保持点击后快速进入后台队列，只刷新当前任务候选和轻量执行范围。
  - 普通点击 `同步任务` 不得默认放大为全量执行扫描、全量历史任务详情刷新或同时同步 Bug。
  - 需要全量扫描时必须由负责人明确要求或另设独立入口。
  - 绿色完成提示必须以真实数据回填为准：只有同步队列结束、任务/Bug 列表刷新完成、同步时间可读后，才允许提示同步完成。
  - 同步刚触发时只能显示同步中或已开始同步，不得提前弹绿色成功提示。
- 数据展示与刷新：
  - AI产物清单、接入扫描、成员产物、资料库库存默认展示平台侧上次扫描缓存。
  - 进入页面、切换页面、切换账号、普通浏览器刷新、打开弹窗、修改UI样式或修改工作台配置时，不得重新抓取 Git、本地路径或共享盘。
  - 只有负责人点击 `刷新库存` 时，才重新扫描 Git、本地路径和共享盘。
  - Git 刷新流程已固定：默认不拉 Git，只有点击 `刷新库存` 才拉取并扫描；失败保留旧缓存。
  - 本地路径和共享盘默认按上次缓存展示；断网、未连接服务器、路径暂时失效或 macOS 重启后未重新挂载时，不得让已扫描内容消失。
  - 修改工作台中的本地路径或共享盘路径，不得影响已经接入扫描的内容展示；新路径只影响后续负责人主动扫描或刷新。
  - 只允许自动处理 `.DS_Store` 等系统元数据导致的缓存仓库污点。
  - 除非负责人明确要求，不得再改 Git 刷新流程、缓存策略、扫描触发逻辑或失败保留策略。
  - 刷新按钮代表主动同步；正常展示读取上次缓存。
  - 扫描失败、网络失败、权限失败、路径未挂载或接口异常时，必须保留当前已有清单和缓存，不得用空结果覆盖。
  - 如果服务端已有平台缓存，组员账号也应能读取同一份缓存数据。
- AI产物 Git 扫描：
  - 点击刷新库存必须抓取最新内容，不只是统计数量变化。
  - 刷新时必须识别新增、修改、删除。
  - 内容变化但数量不变，也必须识别为变化。
  - 变化判断至少考虑 commit、提交时间、标题、描述、`preview` 正文、正文内容、README/Skill内容等有效指纹。
  - Git 扫描若发现产物内容有变更，质量分必须重新计算。
- 接入扫描：
  - Git扫描、本地路径扫描、共享盘扫描都要把结果写入平台缓存，供负责人和组员共同查看。
  - Git 扫描按原本读取方式执行。
  - 本地路径和共享盘扫描时，优先读取目标路径下的一级文件夹名作为产物名，并去重。
  - 如果文件夹内存在可识别的 Skill 文件或二级 Skill 目录，需要摘取内容，并按 Git 扫描到的 Skill/MD 展示方式补齐标题、说明、触发词和路径信息。
  - 如果文件夹内存在可识别的 Skill 文件，Skill 内容要提炼到工作台，并在内容预览中呈现。
  - 已经扫描过且内容无变更的来源，必须保持已扫描状态、上次来源、上次展示内容和上次接入结果，不得因页面刷新或打开弹窗被改成待接入。
  - 来源记录默认使用上一次扫描结果，不实时刷新；只有负责人点击 `刷新库存` 或主动开始扫描，才更新来源记录。
  - 本地路径和共享盘断联时，继续展示上次扫描缓存，不得清空产物列表。
  - 如果负责人后续删除了本地路径或共享盘里的文件夹内容，并在路径可访问时主动刷新，平台必须删除该文件夹产物关联的扫描信息、展示配置、贡献人覆盖、别名和隐藏配置，不得只做前端隐藏。
  - 删除后的路径内容如果后续重新新增，必须按新增接入重新扫描和建档。
  - 已作废或隐藏的产物不计入AI评分，但不得删除源文件或共享盘内容。
  - 本地路径、共享盘、Git 扫描出来的产物，都应保留贡献人调整入口。
  - 修改贡献人只改工作台归属覆盖，不改源文件。
- AI产物调用次数：
  - 调用次数来源于成员同步上报数据和平台累计的产物使用统计，不来自页面假数据。
  - 成员上报的原始明细只作为短期明细缓存；平台必须另存累计调用次数。
  - 原始明细数据最多保留 2 天，超过 2 天要彻底清除，避免使用链接、打开页面或本机运行时卡顿。
  - 清除原始明细不得清零或减少原本的调用次数；新数据来了，只在旧的累计调用次数上增加次数。
  - 调用次数匹配必须同时看产物原名、展示名称和负责人修改后的别名。
  - 如果负责人修改了产物别名，显示调用次数时必须按修改后的别名继续匹配并累加。
  - 路径扫描产物的产物名称和调用别名都必须参与调用次数匹配。
  - 成员上报信息里命中产物名称或任一调用别名时，必须增加该产物调用次数。
  - 工作台上的按钮操作名称如果命中产物名称或任一调用别名，且该操作产生了生成、执行、复用或调用结果，也必须增加该产物调用次数。
- AI产物有效占比：
  - `盛威` 的产物不计算有效占比。
  - 其他人的产物计算有效占比时，固定扣除管理员本人和 `盛威`，不是扣除当前登录账号对应人员。
  - 有效占比必须基于平台真实产物状态、作废/隐藏状态、贡献人覆盖和缓存数据计算，不得使用页面初始假数据。
- AI评分：
  - AI评分必须基于当前平台缓存、产物归属、使用次数、验证/研究同步等真实数据计算。
  - 不得使用页面初始假数据。
  - `产物分计算说明` 必须保留。
  - 作废产物必须从AI评分中排除。
  - 隐藏产物必须从AI评分中排除。
  - 恢复后重新按正常规则计入。
  - 修复AI评分时，只改评分计算和展示相关逻辑，不得连带改动扫描缓存、贡献人覆盖、权限或刷新策略。
- 权限：
  - 负责人可刷新库存、作废/恢复产物、调整贡献人、调整展示管理、修改展示名称或别名。
  - 组员可读取平台缓存。
  - 组员在AI产物清单只显示明细按钮。
  - 不得让组员端因为没有本机扫描能力而显示 `待接入`。
- 已使用/需保持的接口：
  - `/api/project-scan-cache`：读取部门库存缓存。
  - `/api/projects/:id/scan?refresh=1`：负责人主动刷新库存时触发扫描。
- Git自动备份数据约定：
  - 自动备份只提交 Git 工作区内未忽略的项目文件。
  - `.gitignore` 中的 `data/`、`logs/`、`dist/`、`node_modules/`、`.env*`、`outputs/`、`workspace/` 等不得被自动备份提交。
  - 自动备份遇到合并、变基、cherry-pick、远端有本地没有的提交时必须跳过，等待人工处理。
  - 自动备份不做强推。
  - 自动备份不做空提交。
  - 自动备份提交信息使用中文时间戳格式：`自动备份 YYYY-MM-DD HH:mm`。

## 4.禁用写法、已定功能、固定执行命令

- 禁止事项：
  - 禁止未经要求重构业务结构。
  - 禁止顺手修改禅道字段。
  - 禁止扫描失败、路径断联、网络失败或接口异常后用空数据覆盖缓存。
  - 禁止页面进入、普通刷新、打开弹窗或修改工作台配置时自动重新抓取库存。
  - 禁止刷新页面后把 AI产物清单置空或要求重新接入扫描。
  - 禁止把AI产物刷新做成只看数量变化。
  - 禁止组员端因为无扫描权限而看不到平台缓存。
  - 禁止作废产物继续计入AI评分。
  - 禁止隐藏产物继续计入AI评分。
  - 禁止删除源文件或共享盘内容来实现作废或隐藏。
  - 禁止共享盘或本地路径断联时清空已扫描产物。
  - 禁止把接入扫描弹窗做成实时刷新库存入口。
  - 禁止本地路径或共享盘内容删除后只做前端隐藏而不清理扫描关联信息。
  - 禁止成员上报原始明细清理时清零累计调用次数。
  - 禁止有效占比按当前登录账号动态扣除；有效占比固定扣除管理员本人和 `盛威`。
  - 禁止让列表操作栏在响应式布局中消失。
  - 禁止操作按钮换行导致不可见。
  - 禁止任务拖拽依赖禅道同步成功。
  - 禁止禅道同步按钮提前提示绿色成功。
  - 禁止把任务同步和 Bug 同步重新合并为一次点击。
  - 禁止普通任务同步默认扩大为全量历史扫描。
  - 禁止弹窗无限重弹、抢焦点、Dock闪烁。
  - 禁止自动Git备份使用 AI、Codex 或 OpenAI token。
  - 禁止自动Git备份强推或覆盖远端异常状态。
- 固定执行命令：
  - 前端构建验证：
    ```bash
    npm run build
    ```
  - 后端JS语法验证：
    ```bash
    node --check server/server.mjs
    node --check server/scanner.mjs
    ```
  - 启动/重启工作台服务：
    ```bash
    bash scripts/install_platform_launch_agent.sh
    ```
  - 手动执行Git自动备份：
    ```bash
    bash scripts/git-auto-backup.sh
    ```
  - 安装Git自动备份定时任务：
    ```bash
    bash scripts/install_git_auto_backup_launch_agent.sh
    ```
  - 临时修改Git自动备份频率：
    ```bash
    ART_PLATFORM_GIT_BACKUP_INTERVAL=900 bash scripts/install_git_auto_backup_launch_agent.sh
    ```
  - 查看Git状态：
    ```bash
    git status -sb
    ```
  - 推送备份远端：
    ```bash
    git push platform-backup main
    ```
- Git自动备份频率：
  - 默认频率在 `scripts/install_git_auto_backup_launch_agent.sh` 中配置。
  - `1800` 秒表示 30 分钟。
  - 常用值：
    - `600`：10分钟
    - `900`：15分钟
    - `1800`：30分钟
    - `3600`：1小时
    - `7200`：2小时
  - 修改频率后必须重新执行安装脚本才会生效。

- 验证规则：
  - 改后端JS时，至少对被改文件执行 `node --check`。
  - 改前端或样式时，至少执行 `npm run build`。
  - 涉及工作台UI时，优先核对 `/skills/assets`、使用明细抽屉和“新建美术执行”弹窗。
  - 验证缓存逻辑时，不得通过手动刷新库存掩盖默认缓存读取问题。
  - 验证接入扫描逻辑时，必须分别覆盖 Git、本地路径、共享盘路径、路径断联、删除后刷新、重新新增、展示管理勾选和全选。
