# 美术部工作台项目规则

这些规则适用于 `/Users/se7en/ArtProject/platform`。它们来自会话 `019e6421-977c-7180-b4bd-6540291031fd` 中已经反复确认或已经改过的工作台行为。后续修改本项目时，优先遵守这里的项目约束；除非负责人明确要求变更，不要按旧理解回退。

## 负责人视角

- 工作台是美术部门生产工具，不是展示页或营销页；功能修改优先保证任务、AI 产物、成员数据、验证和操作记录准确。
- 未明确要求时，不要顺手调整禅道字段、AI 产物归属、成员权限、扫描来源、刷新策略、任务状态或历史记录。

## 数据与刷新规则

- AI 产物清单、接入扫描、成员产物和资料库库存默认展示平台侧上次扫描缓存。
- 进入页面、切换页面、切换账号、普通浏览器刷新、打开弹窗或修改 UI 样式时，不得重新抓取 Git、本地路径或共享盘。
- 只有负责人点击“刷新库存”时，才重新扫描 Git、本地路径和共享盘，并按新增、修改、删除判断后同步到工作台。
- 扫描失败、网络失败、权限失败或接口异常时，必须保留当前已有清单和缓存，不得用空结果覆盖。
- 列表类数据有刷新按钮时，刷新按钮代表主动同步；正常展示应读取上次缓存，避免每次进入页面数据变化。
- 如果服务端已有平台缓存，组员账号也应能读到同一份缓存数据；不要让组员端因为没有本机扫描能力而显示“待接入”。

## 接入扫描规则

- Git 扫描、本地路径扫描、共享盘扫描都要把结果写入平台缓存，供负责人和组员共同查看。
- 本地路径和共享盘扫描时，优先读取目标路径下的文件夹名作为产物名，并做去重。
- 如果文件夹内存在可识别的 Skill 文件或二级 Skill 目录，需要摘取其内容，按 Git 扫描到的 Skill/MD 展示方式补齐标题、说明、触发词和路径信息。
- 本地路径、共享盘和 Git 扫描出来的产物，都应保留“贡献人”调整入口；修改贡献人只改工作台归属覆盖，不改源文件。
- 已作废或隐藏的产物不计入 AI 评分，但不得删除源文件或共享盘内容。

## AI 评分规则

- AI 评分必须基于当前平台缓存、产物归属、使用次数、验证/研究同步等真实数据计算，不能使用页面初始假数据。
- “产物分计算说明”必须保留，方便负责人理解分数来源。
- 作废产物要从 AI 评分中排除；恢复后再按正常规则计入。
- 修复 AI 评分时，只改评分计算和展示相关逻辑，不得连带改动扫描缓存、贡献人覆盖、权限或刷新策略。

## UI 展示规则

- 工作台界面内容要尽量展示完整。按钮、分段控件、标签、抽屉标题、成员名、产物名和版本记录等关键业务文本不得出现无意义的 `...` 截断。
- 长路径、日志、表格大段文本可以为了布局稳定做受控换行或保留可查看详情入口，但不能让负责人无法判断真实内容。
- “新建美术执行”弹窗里的执行来源和执行范围分段控件必须使用全倒角选中底图，文字完整可见。
- “使用明细”半屏抽屉保持直角表现，布局要紧凑；不要恢复绿色说明框、版本角标或已删除的解释文案。
- 使用明细抽屉通过 `append-to-body` 渲染，相关样式需要同时考虑全局样式和组件内 scoped 样式，避免局部样式不生效。
- 全局视觉修复只处理用户指出的可读性、间距、倒角和截断问题，不要借机改业务结构、字段含义或数据流程。

## 已改过的关键位置

- `server/scanner.mjs`：本地/共享盘扫描支持浅层目录、二级 Skill 目录和常用文件名产物识别。
- `server/server.mjs`：`/api/project-scan-cache` 用于读取部门库存缓存；组员有 `menu.skillList` 时应可读取缓存。
- `src/App.vue`：库存缓存读取、刷新库存失败保留旧数据、贡献人覆盖、AI 评分和产物使用统计都集中在这里。
- `src/views/SkillInventoryView.vue`：AI 产物清单、贡献人调整、使用明细抽屉和成员产物整合入口。
- `src/components/dialogs/RunCreateDialog.vue`：新建美术执行弹窗，执行来源/执行范围分段控件已有专用样式 class。
- `src/styles.scss`：执行弹窗分段控件、使用明细抽屉、AI 评分面板和全局展示样式的兜底规则。

## 验证要求

- 改后端 JS 时，至少对被改文件跑 `node --check`。
- 改前端或样式时，至少跑 `npm run build`。
- 涉及工作台 UI 的变更，优先核对 `/skills/assets`、使用明细抽屉和“新建美术执行”弹窗；不能只看编译通过。
- 验证时不要通过手动刷新库存来掩盖缓存问题。除非测试目标就是刷新库存，否则应先确认默认读取上次缓存是否正常。
