# 项目清理边界说明

本文档用于说明哪些目录可以日常清理，哪些目录必须谨慎处理，避免误删工作台运行数据、执行证据和负责人管理信息。

## 可直接清理

- `.DS_Store`：macOS 自动生成的 Finder 元数据文件。
- `.playwright-cli/`：本地浏览器调试快照和 console 日志。
- 明确为空的临时输出目录，例如已确认无文件的 `outputs/redesign-qa/`。
- 已确认不再分发的旧安装包、旧增量包和旧验证目录。

以上内容不参与前端构建、后端接口、工作台运行数据、AI 产物统计、任务同步或执行记录。

## 可归档后清理

- `outputs/redesign-backups/`：历史样式备份。
- `outputs/style-drafts/`：历史视觉草稿。
- `outputs/*.zip`、`outputs/windows-deploy/`：旧安装包或旧分发包。
- `outputs/member-art-reporter/`、`outputs/member-art-workbench-sync-update/`、`outputs/art-workbench-sync-reporter-skill/`：组员分发包源目录。只有确认已不再给组员安装或已有其它分发来源后才能删除。
- 临时验证目录、截图目录、一次性导出目录。

这类内容一般不影响工作台运行，但可能有追溯价值。默认先移动到 `outputs/archive/`，确认不需要后再删除。

## 不得直接删除

- `data/`：平台本地业务数据、账号角色、任务、AI 产物缓存、调用次数、看板缓存和运行状态。
- `workspace/`：执行工作区、执行日志、执行材料和执行产物。
- `workspace/artifacts/`：美术执行台和 AI 档案需要读取的产物证据。
- `outputs/art-briefs/`：任务中心生成的美术需求摘要和 AI 工作说明。
- `logs/`：平台和自动备份运行日志，清理前需要确认近期排障不再需要。
- `dist/`：生产静态构建结果，只有重新构建并确认服务入口后才能清理。
- `node_modules/`：依赖目录，只有准备重新安装依赖时才能清理。

这些目录和工作台功能存在直接关系，不能按临时文件处理。

## 删除必须走业务入口的内容

- 扫描来源、项目接入记录、AI 产物作废/恢复、执行记录、AI 档案、操作日志、用户、角色等业务数据，必须通过工作台页面或对应 API 删除。
- 不得通过手动删除 JSON、手动删除执行目录或批量清空缓存来替代业务删除入口。
- 删除扫描来源不得顺带删除执行记录、AI 档案明细、执行工作区、执行产物目录或 `usage-counters.json` 累计调用次数。

## 巡检命令

```bash
npm run audit:cleanup
```

这个命令只读输出清理候选、数据体量和外部配置硬编码线索，不会删除、移动或修改任何文件。
