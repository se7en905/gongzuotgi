# 维护优化清单

本文档记录当前已经确认的低影响维护优化项，用于后续按负责人节奏逐步处理。这里不包含会改变工作台功能逻辑的重构项。

## 1. 清理巡检常态化

- 使用 `npm run audit:cleanup` 查看当前可清理项、数据体量和外部配置硬编码线索。
- 巡检脚本只读，不会删除、移动或修改文件。
- 每次手动清理 `outputs/`、`logs/`、`data/restore-backups/` 前，应先跑一次巡检。

## 2. 运行数据说明补全

- `data/` 是业务数据目录，不能手动清空。
- `workspace/` 和 `workspace/artifacts/` 是执行证据目录，删除执行记录必须走工作台入口。
- `outputs/art-briefs/` 是正式任务摘要目录，任务中心可能读取其中的 HTML 和 AI 工作说明。
- `logs/` 用于排障和 LaunchAgent 运行检查，排障窗口结束后再归档或截断。

## 3. 操作日志保留策略

当前后端已有保留机制：

- `AWP_OPERATION_LOG_MAX_ROWS`：操作日志最大行数，默认 `10000`。
- `AWP_DATA_RETENTION_DAYS`：部分增长型记录保留天数，默认 `2`。
- `AWP_DATA_RETENTION_ENABLED`：设置为 `0` 可关闭保留清理。
- `AWP_DATA_RETENTION_CLEANUP_INTERVAL_MS`：服务端定期清理间隔，默认 6 小时。

后续低影响优化建议：

- 在 README 保留上述变量说明。
- 在巡检报告里持续显示 `data/operation-logs.json` 体量和记录数。
- 如需页面可视化，再单独新增“存储健康”页面；不要直接在后台自动删除未知目录。

## 4. 删除入口边界

现有删除操作要继续保持以下边界：

- 删除扫描来源：只删除平台接入记录、同步任务、Bug、备注、验收、摘要和扫描缓存；不删除执行记录、AI 档案、工作区和累计调用次数。
- 删除执行记录或 AI 档案明细：可以清理对应执行工作区和产物目录；不回退 AI 产物累计调用次数。
- 删除操作日志：只删除日志本身；不影响 AI 产物调用次数、有效人数和历史桶。
- 删除平台创建任务：不删除禅道任务，也不删除已产生的执行档案。
- 作废/恢复 AI 产物：只影响工作台展示态和评分计入，不修改源文件。

后续如改删除提示文案，应只强化说明，不改变上述语义。

## 5. 外部配置登记

巡检脚本会列出代码中的外部地址和本机路径线索。当前应先登记，不直接替换默认值，避免影响现有链路。

重点关注：

- 禅道地址。
- Google Sheet ID 和 gid。
- 组员上报内网地址。
- 本机 Codex runtime、Node、Chrome 路径。
- Git 资源仓库地址。

后续优化方式：

- 先在 README 补环境变量说明。
- 再在代码中保持原默认值不变，优先读取环境变量。
- 每次配置化只处理一个链路，并做对应回归。

## 6. 页面入口校准

- README 页面列表要以真实存在的 `src/views/*.vue` 和 `src/components/dialogs/*.vue` 为准。
- 不再写已删除或不存在的旧页面名。
- 新增页面或删除页面后，应同步更新 README。

## 7. 低影响处理顺序

1. 跑 `npm run audit:cleanup`。
2. 清理巡检报告中“可直接清理”的内容。
3. 对“建议负责人确认后再清理”的内容逐项确认。
4. 更新 README 和本维护清单。
5. 暂不处理需要改接口、改页面逻辑或改存储语义的高影响项。
