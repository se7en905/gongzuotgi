平台免 Token 处理方式速查

这份页面用于快速判断工作台里哪些动作可以完全不走 Codex/OpenAI token，哪些只是把消耗转移到执行人本机， 哪些仍然会真正消耗文本 token 或图片额度。核心目的不是解释实现细节，而是帮负责人快速看清“这件事到底耗谁的、现在有没有现成路径”。

项目：美术部工作台 视角：负责人管理口径 重点：免 token / 耗谁的 / 去哪看 文件位置：项目根目录
先看结论

已经完全免 token 的主路径

Git 自动备份、禅道同步、库存扫描、评分重算、维护中心、操作日志、研究同步上报，这些都已是本地脚本或平台接口处理。

最容易误判的点

本机 Worker 不是免 token，它只是把消耗从负责人机器转移到执行人自己的 Codex / Figma / OpenAI 凭据上。

生图要单独看

生图不一定走文本 token，但会走图片额度。默认是执行人本机的 image2 / gpt-image-2，不是平台服务器代跑。

名字带 AI 不等于一定耗 token

AI评分、AI部门看板缓存、AI产物清单刷新库存 当前实现本质上都是本地数据计算、缓存读取或目录扫描。

最明确的免 token 现成能力

Git 自动备份 在项目基线里已明确写死“不经过 Codex 对话、不调用 AI、不消耗 Codex/OpenAI token”。

看这页的顺序

先看下面 4 类卡片，再看“总关系图”，最后到“入口逐项表”里按按钮、页面或脚本直接定位。

完全免 Token

不走 Codex/OpenAI 文本 token，也不依赖负责人本机 AI 对话。

  • Git 自动备份
  • 同步任务 / Bug
  • 刷新库存 / 扫描文件夹
  • 评分重算 / 维护中心

不耗负责人 Token

平台不代跑，但执行人自己的电脑仍会消耗自己的凭据或额度。

  • 本机 Worker 普通执行
  • 本机 Worker Figma 写入
  • 本机 Worker 生图

会耗文本 Token

只要真正跑 codex exec，无论在服务器侧还是 Worker 侧，本质都会消耗 Codex/OpenAI 文本 token。

  • 普通 Codex 执行
  • Figma 写入执行
  • 需要生成报告或多阶段执行的 AI 任务

会耗图片额度

未必走文本 token，但会走图片模型额度或外部生图方案额度。

  • image2 / gpt-image-2 生图
  • 兜底生图方案
  • 同 IP 生图、参考图编辑

负责人判断口诀

先问：有没有进入 codex exec

只要真正进入 codex exec，就不是“免 token”；区别只在于是耗负责人这边，还是耗执行人本机。

名字带 AI，不代表一定耗 token

AI评分、AI部门看板缓存、AI产物清单刷新库存 这些当前实现都属于本地数据处理，不要被名字误导。

生图和文本 token 分开看

生图失败不代表文本 token 有问题；更多时候是执行人本机的图片额度、代理、DNS 或 image2 配置问题。
